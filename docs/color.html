Colors

Use of color in visualization

  1. Distinguish categories (qualitative)
  1. Represent numeric values (sequential)
  1. Represent numeric values (diverging)
  1. Highlight

Using color is good for differentiating between groups, but if you need exact values for numeric or continuous data, color cannot provide that level of detail.

Qualitative scales

Set1 is likely the better choice with small dots, where stronger or darker colors are more useful.

Sequential scales

Some color scales might be more intuitive, such as the inferno scale with temperatures.

Diverging scales

With different color palettes, the area in the middle tends to have lighter colors, but the range that is lightly colored varies between palettes. On the left, we see a larger range, whereas in the centerght, it’s a smaller range. A larger range might not show enough detail with values in the middle, while a smaller range makes it difficult to read. The final color scale on the right is a good in-between.

Highlighting scales

You don’t want a scale meant to highlight a particular category to be too complex or overwhelming, because you want it the highlighted data to stand out. Gray and color, on the left, works well because it is not distracting.

Colorspace

The colorspace package creates some order with scale naming conventions scale_<aesthetic>_<datatype>_<colorscale>(), which contains:

  • <aesthetic>: the name of the aesthetic (fill, color, colour)

  • <datatype>: type of variable plotted (discrete, continuous, binned)

  • <colorscale>: type of color scale (qualitative, sequential, diverging, divergingx)

Which makes functions more clear, such as scale_color_discrete_qualitative().

For example, these do the same thing:

library(colorspace)
p <- ggplot(temps_months, aes(x = month, y = location, fill = mean)) +
  geom_tile(width = 0.95, height = 0.95) +
  coord_fixed(expand = FALSE) + 
  theme_classic()

p +
  scale_fill_distiller(palette = "YlGnBu")
p + 
  scale_fill_continuous_sequential(palette = "YlGnBu", rev = F)

Other Settings

You can also set breaks, labels, and other specifications for the scale:

p + 
  scale_fill_viridis_c(
    name = "temperature",
    breaks = c(25, 50, 75, 100),
    labels = c("25°F", "50°F", "75°F", "100°F")
  )

Common qualitative palettes

Best Practice: Set Colors Manually

Discrete, qualitative scales are often best set manually.

ggplot(temperatures, aes(day_of_year, temperature, color = location)) +
  geom_line(linewidth = 1)

A good option is the ggthemes scale_color_colorblind(), but there are some issues, such as the black standing out almost like a highlight, and the blue and green looking relatively similar.

library(ggthemes)
Warning: package 'ggthemes' was built under R version 4.4.3
ggplot(temperatures, aes(day_of_year, temperature, color = location)) +
  geom_line(linewidth = 1) +
  scale_color_colorblind()

Instead, assigning colors manually is preferable to have full control:

library(ggthemes)
ggplot(temperatures, aes(day_of_year, temperature, color = location)) +
  geom_line(linewidth = 1) +
  scale_color_manual(values = c('Death Valley' = "#E69F00", 
                                'Houston' = "#56B4E9",
                                'San Diego' = "#009E73",
                                'Chicago' = "#F0E442"))

You can access hex codes of palettes using functions like *_pal() for ggthemes, brewer.pal, and paletteer_d/paletteer_c for a full list of color palettes in R using the paletteer package.

Other Palettes

The RColorBrewer package has good qualitative palettes. Use functions scale_fill_brewer(palette = "Set1") and scale_color_brewer(palette = "Set1").

Here, we can see that United operates most flights at EWR, and that delays tend to be longer later in the day. Darker palettes are better, but all qualitative brewer palettes are: Accent, Dark2, Paired, Pastel1, Pastel2, Set1, Set2, Set3.

library(RColorBrewer)
library(tidyverse)
library(nycflights13)

set.seed(123)
nyc <- flights %>% 
  filter(carrier %in% c("AA", "DL", "UA", "WN", "B6", "F9"), origin == "EWR", 
         month == sample(1:12, 1), day == sample(unique(day), 1)) %>% tidyr::drop_na()

p <- ggplot(nyc, aes(x = dep_time, y = dep_delay, color = carrier)) + 
  geom_point(alpha = .7) + 
  theme_classic() +
  scale_y_continuous(limits = c(-20, 150), expand = c(0, 0))

p +
  ggtitle("palette = Set1")  + 
  scale_color_brewer(palette = "Set1")
p +
  ggtitle("palette = Dark2")  + 
  scale_color_brewer(palette = "Dark2")

Common sequential palettes

viridis is a great package for sequential palettes:

p <- ggplot(temps_months, aes(x = month, y = location, fill = mean)) +
  geom_tile(width = 0.95, height = 0.95) +
  coord_fixed(expand = FALSE) + 
  theme_classic() 
p +
  scale_fill_viridis_c() +
  ggtitle("option = 'viridis'")
p + 
  scale_fill_viridis_c(option = "inferno", 
                       begin = .15) + # begin allows you to cut off some of the palette
  ggtitle("option = 'inferno'")

Distiller palettes, which take RColorBrewer scales and make them continuous, are another good option for sequential scales:

p +
  scale_fill_distiller(palette = "YlGnBu") +
  ggtitle("palette = 'YlGnBu'")
p + 
  scale_fill_distiller(palette = "YlOrRd", direction = 1) +
  ggtitle("palette = 'YlOrRd', direction = 1")

Common diverging palettes

All colorspace diverging palettes:

library(fivethirtyeight)
library(sf)

states <- read_sf("tl_2025_us_state") %>% 
  filter(!NAME %in% c("Hawaii", "American Samoa", "United States Virgin Islands", 
                      "District of Columbia", "Puerto Rico", 
                      "Commonwealth of the Northern Mariana Islands", "Guam", 
                      "Alaska")) %>% 
  left_join(partisan_lean_state, by = c("NAME" = "state")) %>% 
  mutate(lean = ifelse(pvi_party == "D", pvi_amount * -1, pvi_amount))

ggplot(data = states, aes(fill = lean)) +
  geom_sf() +
  scale_fill_continuous_diverging(
    name = "Cook Partisan Voting Index",
    palette = "Blue-Red",
    rev = F,
    na.value = "white", 
    limits = c(-50, 50),
    breaks = c(-50, 50),
    labels = c("Lean Democrat", "Lean Republican"),
    # p1 adjusts how chroma should vary, makes chroma increase 
    # quickly near the midpoint (more concave)
    p1 = .6) +
  theme_map() +
  # making the legend look nicer
  theme(legend.position = "inside",
        legend.position.inside = c(.05, .05),
        legend.direction = "horizontal",
        legend.title.position = "top", 
        legend.title = element_text(hjust = .5)) +
  coord_sf(xlim = c(-130, -65), ylim = c(25, 50))
